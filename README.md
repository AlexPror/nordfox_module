# NordFox Module Manager

Desktop-приложение для инженерных проектов NordFox в **КОМПАС-3D**.
Программа автоматизирует массовое обновление параметров и метаданных в сборке, деталях и чертежах.

## Версия

Текущая версия: `0.1.3`

Дорожная карта до релиза 1.0: [docs/roadmap-1.0.md](docs/roadmap-1.0.md).

## Что умеет программа

- Сканирует проект: сборка (`.a3d`), детали (`.m3d`) и чертежи (`.cdw`) — **если в дереве папок есть сборка**.
- **Открытие папки без `.a3d`**: можно указать, например, только каталог с чертежами; блок переменных по сборке будет недоступен, но **экспорт PDF/DWG, штампы и комплектовщик** работают по своим путям.
- Строит форму переменных по данным сборки (переменные деталей в интерфейсе не дублируются).
- Обновляет переменные каскадом для стабильного пересчета геометрии.
- Заполняет новые обозначения/наименования по правилам для ключевых ролей (например, `Р`, `СК`, `СС`).
- Применяет обозначения/наименования в сами документы.
- Синхронизирует вхождения деталей внутри сборки в изолированном subprocess
  (чтобы снизить риск падения UI из-за COM-нестабильности).
- Обновляет штампы чертежей (в т.ч. пакетная нумерация листов, литера, учёт титульного листа; синхронизация настроек листов документа КОМПАС с ручными полями штампа).
- **Комплектовщик чертежей**: порядок строк, переименование, обновление штампов по порядку, **экспорт таблицы «Перечень чертежей комплекта» в `.frw`** (шаблон `table_frw/Table.frw`, см. `table_frw/README.txt`).
- Экспортирует `CDW -> PDF` через локальный сервис, с опцией объединения в один PDF (**порядок листов в общем PDF совпадает с сортировкой по «(лист N)» в имени файла**, как у комплектовщика).
- Экспортирует `CDW -> DWG` напрямую через локальный сервис (без промежуточного DXF).
- Генерирует QR-коды в PNG.
- Поддерживает безопасный режим "Копировать и обновить" для тестов на копии проекта.
- Ведет текстовый и JSON-лог сессии.

## Требования

- Windows 10/11.
- Установленный КОМПАС-3D с доступным COM API.
- Python 3.11+ (рекомендуется та же версия, что в рабочем окружении).
- Зависимости для экспорта:
  - `Flask` (локальный сервис экспорта),
  - `pypdf` 4+ (объединение PDF: используется `PdfWriter`; пакет должен быть в **том же** venv, что и `main.py`).

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
py -3 main.py
```

или через `run.bat`.

## Структура интерфейса

- **Вкладка "Переменные"**
  - отображает переменные сборки по блокам;
  - позволяет вручную менять значения и запускать обновление.
- **Вкладка "Обозн./наимен."**
  - показывает список уникальных документов проекта;
  - позволяет заполнить новые значения по правилам;
  - применяет значения в документы и синхронизирует вхождения в сборке.
- **Вкладка "Штампы"**
  - массово обновляет поля штампов чертежей;
  - отдельная операция автонумерации листов по выбранной папке;
  - путь к чертежам для штампов можно задать независимо от папки проекта.
- **Вкладка "Комплектовщик"**
  - общая папка проекта/чертежей;
  - **«Автонумерация файлов»**: таблица файлов, переименование, штампы по порядку;
  - **«Перечень чертежей»**: предпросмотр таблицы как в `.frw`, параметры FRW, экспорт; титул не входит в перечень; для чертежей узлов в наименование может подставляться строка из ячейки материала штампа (нужен КОМПАС).
- **Вкладка "Экспорт PDF"**
  - **папка с CDW и папка вывода задаются отдельно**; пустое поле источника подставляет папку из «Открыть проект», если она есть;
  - опция объединить все PDF в один файл (имя по умолчанию: «имя_папки_источника - все чертежи.pdf»);
  - если имя объединенного файла введено без расширения, `.pdf` добавляется автоматически.
- **Вкладка "Экспорт DWG"**
  - аналогично PDF: свои пути источника и вывода, без обязательного успешного скана `.a3d`.
- **Правая панель лога**
  - копирование, сохранение и просмотр журналов операций.

## Рекомендуемый рабочий сценарий

### Проект со сборкой (`.a3d`)

1. Открой папку проекта (в дереве должна находиться сборка).
2. Нажми "Пересканировать", если проект изменялся внешне.
3. На вкладке "Переменные" измени нужные значения.
4. Если нужен безопасный прогон — включи режим копии проекта.
5. Запусти обновление переменных.
6. Перейди на вкладку "Обозн./наимен." и нажми заполнение по правилам.
7. При необходимости скорректируй значения вручную.
8. Нажми "Применить новые обозн./наимен.".
9. Повтори "Пересканировать" и проверь, что значения сохранились.
10. Штампы, комплектовщик, экспорт PDF/DWG — по необходимости.

### Только чертежи (без `.a3d` в выбранной папке)

1. «Открыть проект» можно указать на каталог с `.cdw` — появится предупреждение, переменные сборки будут недоступны.
2. Укажи папки на вкладках **Экспорт PDF**, **Экспорт DWG**, **Комплектовщик** или **Штампы** явно при необходимости.

## Почему используется каскадное обновление

В проектах КОМПАС-3D зависимости могут обновляться не полностью при одиночной записи.
Чтобы снизить риск рассинхронизации, программа работает по шагам:

1. Обновление переменных в сборке.
2. Обновление переменных в деталях.
3. Пересборка/сохранение документов.
4. Отдельная синхронизация вхождений деталей внутри сборки.

Это дает более стабильный итог для геометрии и метаданных.

## Логи и диагностика

- Текстовый лог: `logs/nordfox_module.log`.
- JSON-лог сессии: `logs/nordfox_module_YYYY-MM-DD_HH-MM-SS.json`.
- При проблемах со значениями в сборке проверяй блоки:
  - `[Items][Rules]` — какие роли определились и что сгенерировалось;
  - `[Assembly][isolated] stats` — сколько компонентов просмотрено/сопоставлено/обновлено.
- При проблемах экспорта проверяй блоки:
  - сообщения о запуске локального сервиса Flask на `127.0.0.1:5001` (предупреждение Werkzeug о development server для локального использования нормально);
  - ошибки endpoint `/export` (PDF) и `/export_dwg` (DWG),
  - итоговые счетчики `total/exported/failed`.
- Для объединения PDF нужен пакет **`pypdf`** в том же интерпретаторе Python. Если объединение не выполнилось, в окне результата будет блок «Причина / диагностика»; выполни `pip install -r requirements.txt` в активном venv и перезапусти приложение.
- Для DWG на некоторых версиях КОМПАС возможно расхождение статуса COM и фактического результата:
  файл может быть успешно создан, даже если один из вызовов `SaveAs/SaveAs3` вернул ошибку.
  Приложение учитывает это и засчитывает экспорт по факту наличия корректного файла.

## Ограничения

- Работа с COM API КОМПАС-3D зависит от версии КОМПАС и состояния открытых документов.
- На сложных сборках операции могут выполняться заметно дольше.
- Если в проекте сильно нестандартные имена деталей, для корректной автогенерации обозначений
  может потребоваться ручная корректировка.
